# Error Reference

## ERR_4371 — The client terminated the connection before the server finis

**Subsystem:** ingest
**Severity:** error
**Introduced in:** v4.6

### Description

The replica refused the request because its view of the cluster epoch was stale.

### Common causes

- Two writers targeting the same segment without coordination
- A compaction job scheduled during peak write volume
- Clock skew between coordinator nodes

### Resolution

Increase the client read timeout and confirm the proxy is not buffering the full body. If the error persists under low load, inspect the network path rather than the timeout configuration.

### Related configuration

The behaviour of this error is governed by `MAX_INGEST_BUFFER_BYTES`, which defaults to
`30s`. Increasing this value will delay detection of genuinely stuck tasks. See the configuration
reference for interaction with `INGEST_QUORUM_SIZE`.
