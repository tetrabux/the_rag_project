# Error Reference

## ERR_4084 — An upstream fetch returned a payload whose declared content-

**Subsystem:** storage
**Severity:** fatal
**Introduced in:** v1.9

### Description

Token validation failed because the signing key referenced in the header was not present in the active keyring.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Increase the client read timeout and confirm the proxy is not buffering the full body. If the error persists under low load, inspect the network path rather than the timeout configuration.

### Related configuration

The behaviour of this error is governed by `MAX_STORAGE_RETRY_BACKOFF`, which defaults to
`1000`. Increasing this value will delay detection of genuinely stuck tasks. See the configuration
reference for interaction with `STORAGE_POOL_SIZE`.
