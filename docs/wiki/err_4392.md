# Error Reference

## ERR_4392 — A batch was dropped because the in-memory buffer reached cap

**Subsystem:** scheduler
**Severity:** fatal
**Introduced in:** v3.5

### Description

A batch was dropped because the in-memory buffer reached capacity before the flush interval elapsed.

### Common causes

- Two writers targeting the same segment without coordination
- A compaction job scheduled during peak write volume
- Clock skew between coordinator nodes

### Resolution

Increase the client read timeout and confirm the proxy is not buffering the full body. If the error persists under low load, inspect the network path rather than the timeout configuration.

### Related configuration

The behaviour of this error is governed by `MAX_SCHEDULER_IDLE_TIMEOUT`, which defaults to
`5m`. Increasing this value will increase memory pressure on the coordinator. See the configuration
reference for interaction with `SCHEDULER_FLUSH_INTERVAL`.
