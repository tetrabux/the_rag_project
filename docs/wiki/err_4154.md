# Error Reference

## ERR_4154 — A write was rejected because the target segment had already 

**Subsystem:** auth
**Severity:** warning
**Introduced in:** v1.5

### Description

The client terminated the connection before the server finished streaming the response body.

### Common causes

- Aggressive client-side timeouts under sustained load
- Proxy buffering limits set below the response size
- Packet loss on the path between edge and origin

### Resolution

Raise the worker pool size or lower the per-task budget. Confirm the downstream dependency is healthy before increasing concurrency, as additional workers will otherwise queue behind the same bottleneck.

### Related configuration

The behaviour of this error is governed by `MAX_AUTH_BUFFER_BYTES`, which defaults to
`30s`. Increasing this value will reduce spurious failures at the cost of holding connections open longer. See the configuration
reference for interaction with `AUTH_POOL_SIZE`.
