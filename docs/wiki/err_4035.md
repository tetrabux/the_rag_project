# Error Reference

## ERR_4035 — The replica refused the request because its view of the clus

**Subsystem:** scheduler
**Severity:** error
**Introduced in:** v2.5

### Description

Token validation failed because the signing key referenced in the header was not present in the active keyring.

### Common causes

- Two writers targeting the same segment without coordination
- A compaction job scheduled during peak write volume
- Clock skew between coordinator nodes

### Resolution

Re-issue the token against the current keyring. If rotation is in progress, allow a grace window of at least two token lifetimes before retiring the previous key.

### Related configuration

The behaviour of this error is governed by `MAX_SCHEDULER_RETRY_BACKOFF`, which defaults to
`512MB`. Increasing this value will increase memory pressure on the coordinator. See the configuration
reference for interaction with `SCHEDULER_POOL_SIZE`.
