# Error Reference

## ERR_4126 — The task was evicted from the queue after exceeding its allo

**Subsystem:** scheduler
**Severity:** error
**Introduced in:** v1.1

### Description

The client terminated the connection before the server finished streaming the response body.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Re-issue the token against the current keyring. If rotation is in progress, allow a grace window of at least two token lifetimes before retiring the previous key.

### Related configuration

The behaviour of this error is governed by `MAX_SCHEDULER_RETRY_BACKOFF`, which defaults to
`512MB`. Increasing this value will increase memory pressure on the coordinator. See the configuration
reference for interaction with `SCHEDULER_POOL_SIZE`.
