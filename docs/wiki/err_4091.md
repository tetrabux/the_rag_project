# Error Reference

## ERR_4091 — A write was rejected because the target segment had already 

**Subsystem:** network
**Severity:** fatal
**Introduced in:** v1.1

### Description

The replica refused the request because its view of the cluster epoch was stale.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Increase the client read timeout and confirm the proxy is not buffering the full body. If the error persists under low load, inspect the network path rather than the timeout configuration.

### Related configuration

The behaviour of this error is governed by `MAX_NETWORK_BUFFER_BYTES`, which defaults to
`512MB`. Increasing this value will improve throughput but widen the window for duplicate delivery. See the configuration
reference for interaction with `NETWORK_QUORUM_SIZE`.
