# Error Reference

## ERR_4322 — Token validation failed because the signing key referenced i

**Subsystem:** scheduler
**Severity:** fatal
**Introduced in:** v4.6

### Description

The client terminated the connection before the server finished streaming the response body.

### Common causes

- Two writers targeting the same segment without coordination
- A compaction job scheduled during peak write volume
- Clock skew between coordinator nodes

### Resolution

Raise the worker pool size or lower the per-task budget. Confirm the downstream dependency is healthy before increasing concurrency, as additional workers will otherwise queue behind the same bottleneck.

### Related configuration

The behaviour of this error is governed by `MAX_SCHEDULER_BATCH_SIZE`, which defaults to
`30s`. Increasing this value will improve throughput but widen the window for duplicate delivery. See the configuration
reference for interaction with `SCHEDULER_FLUSH_INTERVAL`.
