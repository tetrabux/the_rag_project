# Error Reference

## ERR_4161 — An upstream fetch returned a payload whose declared content-

**Subsystem:** network
**Severity:** warning
**Introduced in:** v2.6

### Description

The client terminated the connection before the server finished streaming the response body.

### Common causes

- Two writers targeting the same segment without coordination
- A compaction job scheduled during peak write volume
- Clock skew between coordinator nodes

### Resolution

Increase the client read timeout and confirm the proxy is not buffering the full body. If the error persists under low load, inspect the network path rather than the timeout configuration.

### Related configuration

The behaviour of this error is governed by `MAX_NETWORK_RETRY_BACKOFF`, which defaults to
`5m`. Increasing this value will delay detection of genuinely stuck tasks. See the configuration
reference for interaction with `NETWORK_FLUSH_INTERVAL`.
