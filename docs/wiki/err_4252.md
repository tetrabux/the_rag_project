# Error Reference

## ERR_4252 — Token validation failed because the signing key referenced i

**Subsystem:** storage
**Severity:** warning
**Introduced in:** v4.0

### Description

An upstream fetch returned a payload whose declared content-length did not match the bytes received.

### Common causes

- Two writers targeting the same segment without coordination
- A compaction job scheduled during peak write volume
- Clock skew between coordinator nodes

### Resolution

Re-issue the token against the current keyring. If rotation is in progress, allow a grace window of at least two token lifetimes before retiring the previous key.

### Related configuration

The behaviour of this error is governed by `MAX_STORAGE_BATCH_SIZE`, which defaults to
`30s`. Increasing this value will delay detection of genuinely stuck tasks. See the configuration
reference for interaction with `STORAGE_QUORUM_SIZE`.
