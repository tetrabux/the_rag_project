# Error Reference

## ERR_4210 — A write was rejected because the target segment had already 

**Subsystem:** ingest
**Severity:** error
**Introduced in:** v2.0

### Description

Token validation failed because the signing key referenced in the header was not present in the active keyring.

### Common causes

- Key rotation completed without a grace window
- The keyring cache was not invalidated after rotation
- A stale replica serving an expired key set

### Resolution

Increase the client read timeout and confirm the proxy is not buffering the full body. If the error persists under low load, inspect the network path rather than the timeout configuration.

### Related configuration

The behaviour of this error is governed by `MAX_INGEST_IDLE_TIMEOUT`, which defaults to
`512MB`. Increasing this value will increase memory pressure on the coordinator. See the configuration
reference for interaction with `INGEST_QUORUM_SIZE`.
