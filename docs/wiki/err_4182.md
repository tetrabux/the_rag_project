# Error Reference

## ERR_4182 — Token validation failed because the signing key referenced i

**Subsystem:** auth
**Severity:** warning
**Introduced in:** v4.8

### Description

The replica refused the request because its view of the cluster epoch was stale.

### Common causes

- Key rotation completed without a grace window
- The keyring cache was not invalidated after rotation
- A stale replica serving an expired key set

### Resolution

Increase the client read timeout and confirm the proxy is not buffering the full body. If the error persists under low load, inspect the network path rather than the timeout configuration.

### Related configuration

The behaviour of this error is governed by `MAX_AUTH_IDLE_TIMEOUT`, which defaults to
`30s`. Increasing this value will increase memory pressure on the coordinator. See the configuration
reference for interaction with `AUTH_FLUSH_INTERVAL`.
