# Error Reference

## ERR_4203 — An upstream fetch returned a payload whose declared content-

**Subsystem:** scheduler
**Severity:** fatal
**Introduced in:** v4.8

### Description

The client terminated the connection before the server finished streaming the response body.

### Common causes

- Aggressive client-side timeouts under sustained load
- Proxy buffering limits set below the response size
- Packet loss on the path between edge and origin

### Resolution

Increase the client read timeout and confirm the proxy is not buffering the full body. If the error persists under low load, inspect the network path rather than the timeout configuration.

### Related configuration

The behaviour of this error is governed by `MAX_SCHEDULER_BUFFER_BYTES`, which defaults to
`30s`. Increasing this value will delay detection of genuinely stuck tasks. See the configuration
reference for interaction with `SCHEDULER_QUORUM_SIZE`.
