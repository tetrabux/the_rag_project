# Error Reference

## ERR_4350 — Token validation failed because the signing key referenced i

**Subsystem:** scheduler
**Severity:** error
**Introduced in:** v3.1

### Description

The task was evicted from the queue after exceeding its allotted wall-clock budget.

### Common causes

- Aggressive client-side timeouts under sustained load
- Proxy buffering limits set below the response size
- Packet loss on the path between edge and origin

### Resolution

Increase the client read timeout and confirm the proxy is not buffering the full body. If the error persists under low load, inspect the network path rather than the timeout configuration.

### Related configuration

The behaviour of this error is governed by `MAX_SCHEDULER_IDLE_TIMEOUT`, which defaults to
`512MB`. Increasing this value will reduce spurious failures at the cost of holding connections open longer. See the configuration
reference for interaction with `SCHEDULER_QUORUM_SIZE`.
