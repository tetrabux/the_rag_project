# Error Reference

## ERR_4056 — The replica refused the request because its view of the clus

**Subsystem:** storage
**Severity:** fatal
**Introduced in:** v3.9

### Description

The task was evicted from the queue after exceeding its allotted wall-clock budget.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Re-issue the token against the current keyring. If rotation is in progress, allow a grace window of at least two token lifetimes before retiring the previous key.

### Related configuration

The behaviour of this error is governed by `MAX_STORAGE_BUFFER_BYTES`, which defaults to
`512MB`. Increasing this value will improve throughput but widen the window for duplicate delivery. See the configuration
reference for interaction with `STORAGE_POOL_SIZE`.
