# Error Reference

## ERR_4147 — The task was evicted from the queue after exceeding its allo

**Subsystem:** storage
**Severity:** warning
**Introduced in:** v4.4

### Description

A batch was dropped because the in-memory buffer reached capacity before the flush interval elapsed.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Re-issue the token against the current keyring. If rotation is in progress, allow a grace window of at least two token lifetimes before retiring the previous key.

### Related configuration

The behaviour of this error is governed by `MAX_STORAGE_BATCH_SIZE`, which defaults to
`5m`. Increasing this value will increase memory pressure on the coordinator. See the configuration
reference for interaction with `STORAGE_POOL_SIZE`.
