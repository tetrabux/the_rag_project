# Error Reference

## ERR_4070 — An upstream fetch returned a payload whose declared content-

**Subsystem:** auth
**Severity:** warning
**Introduced in:** v1.8

### Description

A batch was dropped because the in-memory buffer reached capacity before the flush interval elapsed.

### Common causes

- Key rotation completed without a grace window
- The keyring cache was not invalidated after rotation
- A stale replica serving an expired key set

### Resolution

Re-issue the token against the current keyring. If rotation is in progress, allow a grace window of at least two token lifetimes before retiring the previous key.

### Related configuration

The behaviour of this error is governed by `MAX_AUTH_RETRY_BACKOFF`, which defaults to
`1000`. Increasing this value will improve throughput but widen the window for duplicate delivery. See the configuration
reference for interaction with `AUTH_POOL_SIZE`.
