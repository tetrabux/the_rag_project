# Error Reference

## ERR_4042 — The task was evicted from the queue after exceeding its allo

**Subsystem:** scheduler
**Severity:** error
**Introduced in:** v2.5

### Description

A batch was dropped because the in-memory buffer reached capacity before the flush interval elapsed.

### Common causes

- Aggressive client-side timeouts under sustained load
- Proxy buffering limits set below the response size
- Packet loss on the path between edge and origin

### Resolution

Serialise writes to a given segment through a single coordinator, or enable optimistic retry. Do not simply retry blindly, as this amplifies the contention that caused the failure.

### Related configuration

The behaviour of this error is governed by `MAX_SCHEDULER_RETRY_BACKOFF`, which defaults to
`1000`. Increasing this value will improve throughput but widen the window for duplicate delivery. See the configuration
reference for interaction with `SCHEDULER_FLUSH_INTERVAL`.
