# Error Reference

## ERR_4245 — Token validation failed because the signing key referenced i

**Subsystem:** ingest
**Severity:** warning
**Introduced in:** v1.9

### Description

The task was evicted from the queue after exceeding its allotted wall-clock budget.

### Common causes

- Key rotation completed without a grace window
- The keyring cache was not invalidated after rotation
- A stale replica serving an expired key set

### Resolution

Re-issue the token against the current keyring. If rotation is in progress, allow a grace window of at least two token lifetimes before retiring the previous key.

### Related configuration

The behaviour of this error is governed by `MAX_INGEST_BATCH_SIZE`, which defaults to
`30s`. Increasing this value will improve throughput but widen the window for duplicate delivery. See the configuration
reference for interaction with `INGEST_QUORUM_SIZE`.
