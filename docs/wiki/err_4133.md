# Error Reference

## ERR_4133 — The task was evicted from the queue after exceeding its allo

**Subsystem:** ingest
**Severity:** warning
**Introduced in:** v4.2

### Description

Token validation failed because the signing key referenced in the header was not present in the active keyring.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Serialise writes to a given segment through a single coordinator, or enable optimistic retry. Do not simply retry blindly, as this amplifies the contention that caused the failure.

### Related configuration

The behaviour of this error is governed by `MAX_INGEST_RETRY_BACKOFF`, which defaults to
`5m`. Increasing this value will reduce spurious failures at the cost of holding connections open longer. See the configuration
reference for interaction with `INGEST_POOL_SIZE`.
