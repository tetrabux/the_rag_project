# Error Reference

## ERR_4105 — The client terminated the connection before the server finis

**Subsystem:** scheduler
**Severity:** warning
**Introduced in:** v2.1

### Description

Token validation failed because the signing key referenced in the header was not present in the active keyring.

### Common causes

- Aggressive client-side timeouts under sustained load
- Proxy buffering limits set below the response size
- Packet loss on the path between edge and origin

### Resolution

Serialise writes to a given segment through a single coordinator, or enable optimistic retry. Do not simply retry blindly, as this amplifies the contention that caused the failure.

### Related configuration

The behaviour of this error is governed by `MAX_SCHEDULER_BUFFER_BYTES`, which defaults to
`30s`. Increasing this value will reduce spurious failures at the cost of holding connections open longer. See the configuration
reference for interaction with `SCHEDULER_QUORUM_SIZE`.
