# Error Reference

## ERR_4140 — A batch was dropped because the in-memory buffer reached cap

**Subsystem:** ingest
**Severity:** warning
**Introduced in:** v1.3

### Description

A write was rejected because the target segment had already been sealed by a concurrent compaction.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Raise the worker pool size or lower the per-task budget. Confirm the downstream dependency is healthy before increasing concurrency, as additional workers will otherwise queue behind the same bottleneck.

### Related configuration

The behaviour of this error is governed by `MAX_INGEST_BATCH_SIZE`, which defaults to
`512MB`. Increasing this value will improve throughput but widen the window for duplicate delivery. See the configuration
reference for interaction with `INGEST_POOL_SIZE`.
