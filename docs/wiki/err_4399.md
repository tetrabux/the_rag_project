# Error Reference

## ERR_4399 — An upstream fetch returned a payload whose declared content-

**Subsystem:** ingest
**Severity:** fatal
**Introduced in:** v4.5

### Description

The client terminated the connection before the server finished streaming the response body.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Re-issue the token against the current keyring. If rotation is in progress, allow a grace window of at least two token lifetimes before retiring the previous key.

### Related configuration

The behaviour of this error is governed by `MAX_INGEST_BUFFER_BYTES`, which defaults to
`5m`. Increasing this value will increase memory pressure on the coordinator. See the configuration
reference for interaction with `INGEST_FLUSH_INTERVAL`.
