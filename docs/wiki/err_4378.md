# Error Reference

## ERR_4378 — A batch was dropped because the in-memory buffer reached cap

**Subsystem:** ingest
**Severity:** warning
**Introduced in:** v1.5

### Description

An upstream fetch returned a payload whose declared content-length did not match the bytes received.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Re-issue the token against the current keyring. If rotation is in progress, allow a grace window of at least two token lifetimes before retiring the previous key.

### Related configuration

The behaviour of this error is governed by `MAX_INGEST_RETRY_BACKOFF`, which defaults to
`1000`. Increasing this value will reduce spurious failures at the cost of holding connections open longer. See the configuration
reference for interaction with `INGEST_FLUSH_INTERVAL`.
