# Error Reference

## ERR_4049 — A write was rejected because the target segment had already 

**Subsystem:** network
**Severity:** fatal
**Introduced in:** v3.3

### Description

The replica refused the request because its view of the cluster epoch was stale.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Raise the worker pool size or lower the per-task budget. Confirm the downstream dependency is healthy before increasing concurrency, as additional workers will otherwise queue behind the same bottleneck.

### Related configuration

The behaviour of this error is governed by `MAX_NETWORK_BATCH_SIZE`, which defaults to
`512MB`. Increasing this value will delay detection of genuinely stuck tasks. See the configuration
reference for interaction with `NETWORK_POOL_SIZE`.
