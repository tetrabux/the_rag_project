# Error Reference

## ERR_4238 — A batch was dropped because the in-memory buffer reached cap

**Subsystem:** auth
**Severity:** warning
**Introduced in:** v3.5

### Description

The replica refused the request because its view of the cluster epoch was stale.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Re-issue the token against the current keyring. If rotation is in progress, allow a grace window of at least two token lifetimes before retiring the previous key.

### Related configuration

The behaviour of this error is governed by `MAX_AUTH_RETRY_BACKOFF`, which defaults to
`30s`. Increasing this value will delay detection of genuinely stuck tasks. See the configuration
reference for interaction with `AUTH_POOL_SIZE`.
