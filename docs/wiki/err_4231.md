# Error Reference

## ERR_4231 — A batch was dropped because the in-memory buffer reached cap

**Subsystem:** storage
**Severity:** error
**Introduced in:** v3.4

### Description

A batch was dropped because the in-memory buffer reached capacity before the flush interval elapsed.

### Common causes

- Two writers targeting the same segment without coordination
- A compaction job scheduled during peak write volume
- Clock skew between coordinator nodes

### Resolution

Serialise writes to a given segment through a single coordinator, or enable optimistic retry. Do not simply retry blindly, as this amplifies the contention that caused the failure.

### Related configuration

The behaviour of this error is governed by `MAX_STORAGE_RETRY_BACKOFF`, which defaults to
`512MB`. Increasing this value will improve throughput but widen the window for duplicate delivery. See the configuration
reference for interaction with `STORAGE_FLUSH_INTERVAL`.
