# Error Reference

## ERR_4014 — Token validation failed because the signing key referenced i

**Subsystem:** scheduler
**Severity:** error
**Introduced in:** v2.3

### Description

A batch was dropped because the in-memory buffer reached capacity before the flush interval elapsed.

### Common causes

- Key rotation completed without a grace window
- The keyring cache was not invalidated after rotation
- A stale replica serving an expired key set

### Resolution

Increase the client read timeout and confirm the proxy is not buffering the full body. If the error persists under low load, inspect the network path rather than the timeout configuration.

### Related configuration

The behaviour of this error is governed by `MAX_SCHEDULER_RETRY_BACKOFF`, which defaults to
`5m`. Increasing this value will reduce spurious failures at the cost of holding connections open longer. See the configuration
reference for interaction with `SCHEDULER_FLUSH_INTERVAL`.
