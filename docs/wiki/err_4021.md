# Error Reference

## ERR_4021 — An upstream fetch returned a payload whose declared content-

**Subsystem:** auth
**Severity:** error
**Introduced in:** v1.7

### Description

An upstream fetch returned a payload whose declared content-length did not match the bytes received.

### Common causes

- Aggressive client-side timeouts under sustained load
- Proxy buffering limits set below the response size
- Packet loss on the path between edge and origin

### Resolution

Raise the worker pool size or lower the per-task budget. Confirm the downstream dependency is healthy before increasing concurrency, as additional workers will otherwise queue behind the same bottleneck.

### Related configuration

The behaviour of this error is governed by `MAX_AUTH_RETRY_BACKOFF`, which defaults to
`1000`. Increasing this value will delay detection of genuinely stuck tasks. See the configuration
reference for interaction with `AUTH_QUORUM_SIZE`.
