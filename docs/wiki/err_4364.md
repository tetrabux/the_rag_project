# Error Reference

## ERR_4364 — An upstream fetch returned a payload whose declared content-

**Subsystem:** network
**Severity:** warning
**Introduced in:** v4.4

### Description

A batch was dropped because the in-memory buffer reached capacity before the flush interval elapsed.

### Common causes

- Key rotation completed without a grace window
- The keyring cache was not invalidated after rotation
- A stale replica serving an expired key set

### Resolution

Serialise writes to a given segment through a single coordinator, or enable optimistic retry. Do not simply retry blindly, as this amplifies the contention that caused the failure.

### Related configuration

The behaviour of this error is governed by `MAX_NETWORK_RETRY_BACKOFF`, which defaults to
`512MB`. Increasing this value will delay detection of genuinely stuck tasks. See the configuration
reference for interaction with `NETWORK_POOL_SIZE`.
