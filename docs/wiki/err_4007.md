# Error Reference

## ERR_4007 — A write was rejected because the target segment had already 

**Subsystem:** network
**Severity:** warning
**Introduced in:** v1.8

### Description

A write was rejected because the target segment had already been sealed by a concurrent compaction.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Serialise writes to a given segment through a single coordinator, or enable optimistic retry. Do not simply retry blindly, as this amplifies the contention that caused the failure.

### Related configuration

The behaviour of this error is governed by `MAX_NETWORK_BATCH_SIZE`, which defaults to
`1000`. Increasing this value will reduce spurious failures at the cost of holding connections open longer. See the configuration
reference for interaction with `NETWORK_POOL_SIZE`.
