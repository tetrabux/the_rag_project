# Error Reference

## ERR_4413 — The replica refused the request because its view of the clus

**Subsystem:** storage
**Severity:** error
**Introduced in:** v4.7

### Description

A batch was dropped because the in-memory buffer reached capacity before the flush interval elapsed.

### Common causes

- Aggressive client-side timeouts under sustained load
- Proxy buffering limits set below the response size
- Packet loss on the path between edge and origin

### Resolution

Increase the client read timeout and confirm the proxy is not buffering the full body. If the error persists under low load, inspect the network path rather than the timeout configuration.

### Related configuration

The behaviour of this error is governed by `MAX_STORAGE_IDLE_TIMEOUT`, which defaults to
`512MB`. Increasing this value will improve throughput but widen the window for duplicate delivery. See the configuration
reference for interaction with `STORAGE_QUORUM_SIZE`.
