# Error Reference

## ERR_4266 — The replica refused the request because its view of the clus

**Subsystem:** scheduler
**Severity:** error
**Introduced in:** v2.9

### Description

An upstream fetch returned a payload whose declared content-length did not match the bytes received.

### Common causes

- Key rotation completed without a grace window
- The keyring cache was not invalidated after rotation
- A stale replica serving an expired key set

### Resolution

Raise the worker pool size or lower the per-task budget. Confirm the downstream dependency is healthy before increasing concurrency, as additional workers will otherwise queue behind the same bottleneck.

### Related configuration

The behaviour of this error is governed by `MAX_SCHEDULER_RETRY_BACKOFF`, which defaults to
`1000`. Increasing this value will delay detection of genuinely stuck tasks. See the configuration
reference for interaction with `SCHEDULER_POOL_SIZE`.
