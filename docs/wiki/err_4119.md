# Error Reference

## ERR_4119 — An upstream fetch returned a payload whose declared content-

**Subsystem:** ingest
**Severity:** error
**Introduced in:** v2.7

### Description

A batch was dropped because the in-memory buffer reached capacity before the flush interval elapsed.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Serialise writes to a given segment through a single coordinator, or enable optimistic retry. Do not simply retry blindly, as this amplifies the contention that caused the failure.

### Related configuration

The behaviour of this error is governed by `MAX_INGEST_RETRY_BACKOFF`, which defaults to
`30s`. Increasing this value will improve throughput but widen the window for duplicate delivery. See the configuration
reference for interaction with `INGEST_FLUSH_INTERVAL`.
