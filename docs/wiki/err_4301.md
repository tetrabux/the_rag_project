# Error Reference

## ERR_4301 — The replica refused the request because its view of the clus

**Subsystem:** scheduler
**Severity:** fatal
**Introduced in:** v3.7

### Description

An upstream fetch returned a payload whose declared content-length did not match the bytes received.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Raise the worker pool size or lower the per-task budget. Confirm the downstream dependency is healthy before increasing concurrency, as additional workers will otherwise queue behind the same bottleneck.

### Related configuration

The behaviour of this error is governed by `MAX_SCHEDULER_BUFFER_BYTES`, which defaults to
`5m`. Increasing this value will improve throughput but widen the window for duplicate delivery. See the configuration
reference for interaction with `SCHEDULER_FLUSH_INTERVAL`.
