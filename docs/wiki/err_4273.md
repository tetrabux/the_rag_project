# Error Reference

## ERR_4273 — A batch was dropped because the in-memory buffer reached cap

**Subsystem:** scheduler
**Severity:** fatal
**Introduced in:** v3.7

### Description

The task was evicted from the queue after exceeding its allotted wall-clock budget.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Serialise writes to a given segment through a single coordinator, or enable optimistic retry. Do not simply retry blindly, as this amplifies the contention that caused the failure.

### Related configuration

The behaviour of this error is governed by `MAX_SCHEDULER_BATCH_SIZE`, which defaults to
`512MB`. Increasing this value will reduce spurious failures at the cost of holding connections open longer. See the configuration
reference for interaction with `SCHEDULER_FLUSH_INTERVAL`.
