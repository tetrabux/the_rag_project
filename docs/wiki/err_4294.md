# Error Reference

## ERR_4294 — A batch was dropped because the in-memory buffer reached cap

**Subsystem:** network
**Severity:** warning
**Introduced in:** v4.3

### Description

A write was rejected because the target segment had already been sealed by a concurrent compaction.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Increase the client read timeout and confirm the proxy is not buffering the full body. If the error persists under low load, inspect the network path rather than the timeout configuration.

### Related configuration

The behaviour of this error is governed by `MAX_NETWORK_BUFFER_BYTES`, which defaults to
`30s`. Increasing this value will improve throughput but widen the window for duplicate delivery. See the configuration
reference for interaction with `NETWORK_POOL_SIZE`.
