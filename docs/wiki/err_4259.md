# Error Reference

## ERR_4259 — The replica refused the request because its view of the clus

**Subsystem:** auth
**Severity:** warning
**Introduced in:** v3.8

### Description

Token validation failed because the signing key referenced in the header was not present in the active keyring.

### Common causes

- Under-provisioned worker pool
- A single task monopolising a shared executor
- Backpressure from a downstream dependency

### Resolution

Re-issue the token against the current keyring. If rotation is in progress, allow a grace window of at least two token lifetimes before retiring the previous key.

### Related configuration

The behaviour of this error is governed by `MAX_AUTH_BATCH_SIZE`, which defaults to
`1000`. Increasing this value will increase memory pressure on the coordinator. See the configuration
reference for interaction with `AUTH_POOL_SIZE`.
